What we're building

Day 1 is about foundations. We're building a fully functional, multi-turn chatbot that streams Claude's responses token by token — so users see output immediately, just like ChatGPT's typing effect — without any JavaScript or WebSocket boilerplate. Just Python.

By the end you'll have a working app, and more importantly, you'll deeply understand three primitives that every AI app in this series will rely on: session state, streaming, and the messages API format.

Key concepts

Session State

Persists Python variables across Streamlit reruns — essential for chat history.

Streaming

Yields tokens as they're generated. Dramatically better UX than waiting for the full response.

Messages Format

Claude's API uses an alternating user/assistant list. We append to it on every turn.

System Prompt

A top-level instruction that shapes Claude's personality and behaviour for the whole session.

Building it step by step

1. Page config and imports

pythonimport streamlit as st
import anthropic

st.set_page_config(
    page_title="Day 1 — AI Chatbot",
    page_icon="🤖",
    layout="centered",
)

st.title("Day 1: AI Chatbot")
st.caption("Powered by Claude · 30 Days of Streamlit: AI Edition")

2. Initialize the client and session state

pythonclient = anthropic.Anthropic()

if "messages" not in st.session_state:
    st.session_state.messages = []
Why the if guard?
Streamlit reruns your entire script on every interaction. Without the guard, messages would reset to [] every time the user types — wiping the chat history. The if check means: "only initialise once."

3. Sidebar controls

pythonwith st.sidebar():
    system_prompt = st.text_area(
        "System prompt",
        value="You are a helpful, friendly AI assistant. Be concise and clear.",
        height=120,
    )
    if st.button("Clear chat", use_container_width=True):
        st.session_state.messages = []
        st.rerun()

4. Render chat history

pythonfor message in st.session_state.messages:
    with st.chat_message(message["role"]):
        st.markdown(message["content"])
Stop and think
This loop runs on every rerun. That's intentional — it replays the full conversation so the UI always reflects the current state of st.session_state.messages.

5. Handle input and stream the response

pythonif prompt := st.chat_input("Message Claude..."):
    # Add user message to history and display it
    st.session_state.messages.append({"role": "user", "content": prompt})
    with st.chat_message("user"):
        st.markdown(prompt)

    # Stream Claude's response
    with st.chat_message("assistant"):
        placeholder = st.empty()
        full_response = ""

        with client.messages.stream(
            model="claude-opus-4-6",
            max_tokens=1024,
            system=system_prompt,
            messages=st.session_state.messages,
        ) as stream:
            for chunk in stream.text_stream:
                full_response += chunk
                placeholder.markdown(full_response + "▌")

        placeholder.markdown(full_response)

    # Save assistant turn to history
    st.session_state.messages.append({
        "role": "assistant",
        "content": full_response,
    })
Two things worth noting
  • st.empty() creates a single slot in the UI that we overwrite on every chunk — this is how the "typing" animation works without flickering.
  • The ▌ cursor character is appended during streaming and removed once streaming is complete, giving a natural terminal feel.

Run the app

bashstreamlit run app.py

Open http://localhost:8501 in your browser. Type a message and watch Claude stream back a response in real time.

Testing multi-turn memory

To verify that conversation context is working correctly, try this sequence:

  1. Send: "My name is Alex."
  2. Send: "What's my name?"
  3. Send: "What was the very first thing I told you?"

Claude should answer all three correctly. This works because st.session_state.messages is passed in full on every API call — Claude sees the entire conversation history each time.

What we learned

  • Session state is the backbone of any stateful Streamlit app.
  • Streaming with client.messages.stream() and st.empty() gives a native chat feel with minimal code.
  • The messages list format (alternating user/assistant dicts) is the contract every Claude app must follow.
  • A system prompt is the cleanest way to control tone and behaviour without touching the conversation history.
